## 2. 核心のアイデア: 局所混雑度

グラフ上のエッジを「残すか消すか」判断するとき、何を基準にすべきでしょうか。

一つの着眼点は**混雑度**です。あるノードに10本のエッジがあるとき、そのうち1本が消えても情報の大きな損失にはなりません。しかし、ノードに2本しかエッジがないとき、1本の消失は情報の半分を失うことを意味します。

この考え方をエッジ単位に適用します。エッジ(u, v)の局所混雑度Cを、両端ノードの次数の和として定義します。

```
C = deg(u) + deg(v)
```

この定義の重要な性質は、**エッジの両端ノードの情報だけで計算できる**ことです。グラフ全体を走査する必要がありません。

---

## 3. 数理モデル

### 3.1 減衰率

局所混雑度Cを使って、エッジの重みがどれだけ速く減衰するかを決定します。

```
λ(C) = β × (1 + γ × C^α)
```

| パラメータ | 推奨値 | 意味 |
|---|---|---|
| β (beta) | 0.05 | 基底減衰率。全てのエッジに適用される最低限の減衰速度 |
| γ (gamma) | 0.5 | 混雑感度。混雑度がどれだけ減衰を加速するか |
| α (alpha) | 2.0 | べき乗指数。混雑度に対する非線形応答の強さ |

αが重要なパラメータです。α = 2の場合、混雑度が2倍になると加速項は4倍になります。混雑した領域での整理が急激に加速するということです。

### 3.2 重み更新式

減衰率λを用いて、エッジの重みを指数関数的に減衰させます。

```
w(t + dt) = w(t) × exp(-λ(C) × dt)
```

指数関数的減衰を採用する理由は2つです。

1. **連続性**: 重みが不連続にジャンプしない（突然の変化がない）
2. **正値性**: exp(-x) > 0 なので、重みは常に正値を保つ

### 3.3 剪定

重みが閾値θを下回ったとき、そのエッジを除去します。

```
if w < θ (= 0.1):  エッジを除去
```

エッジが全て除去されて孤立したノードは、希少性保護の対象でなければ除去されます。

## 4. 数値例: 疎な領域 vs 混雑領域

具体的な数値で挙動を確認しましょう。

### 疎な領域 (C = 4)

両端ノードの次数がそれぞれ2のエッジです。

```python
C = 2 + 2 = 4
λ = 0.05 × (1 + 0.5 × 4^2) = 0.05 × (1 + 8) = 0.45

# 5ステップ後の重み（初期値1.0）
w = 1.0 × exp(-0.45 × 5) = 1.0 × exp(-2.25) ≈ 0.105
# → まだ閾値(0.1)を上回る。生存。
```

### 混雑領域 (C = 20)

両端ノードの次数がそれぞれ10のエッジです。

```python
C = 10 + 10 = 20
λ = 0.05 × (1 + 0.5 × 20^2) = 0.05 × (1 + 200) = 10.05

# 1ステップ後の重み（初期値1.0）
w = 1.0 × exp(-10.05 × 1) ≈ 0.0000435
# → 1ステップで実質消滅。
```

疎な領域のエッジは5ステップ後でもかろうじて生き残りますが、混雑領域のエッジは1ステップで消滅します。これが**適応的**な整理です。

### 重み減衰の比較表

初期重み1.0、dt=1.0での各ステップの重みを比較します。

| ステップ | C=4 (疎) | C=10 (中) | C=20 (混雑) |
|:---:|:---:|:---:|:---:|
| 0 | 1.000 | 1.000 | 1.000 |
| 1 | 0.638 | 0.0067 | 0.0000 |
| 2 | 0.407 | 0.0000 | - |
| 3 | 0.259 | - | - |
| 5 | 0.105 | - | - |
| 6 | 0.067 | - | - |

C=4のエッジは6ステップまで生存、C=10は2ステップで消滅、C=20は1ステップで消滅。混雑度の差がこれほど劇的な違いを生みます。

## 5. 計算量の分析

### 5.1 単一エッジの処理

エッジ(u, v)の処理に必要な情報は以下のみです。

1. `deg(u)`: ノードuの次数 — ノードに保持された局所統計から O(1)
2. `deg(v)`: ノードvの次数 — 同上
3. `w`: エッジの現在の重み — エッジに保持
4. `β, γ, α, dt, θ`: 定数

全て定数時間で取得可能であり、**1エッジあたりO(1)** で処理が完結します。

### 5.2 グラフ全体の処理

全エッジを1回処理するコストは O(|E|) です。通常のグラフアルゴリズム（BFS、PageRank等）が O(|V| + |E|) のグラフ走査を前提とするのに対し、代謝制御は個々のエッジを独立に処理できます。

### 5.3 分散実行

エッジ間の依存関係がないため、グラフをパーティション分割して各処理主体が独立に代謝制御を実行できます。処理主体間で共有する必要がある情報は、メタ制御のための平均次数kAvg程度です。

## 6. 実装

リファレンス実装（Python）のコア部分を示します。

```python
def decay_rate(C: float, alpha: float = 2.0,
               beta: float = 0.05, gamma: float = 0.5) -> float:
    """局所混雑度Cに基づく減衰率を算出"""
    return beta * (1.0 + gamma * (C ** alpha))

def update_weight(w: float, lam: float, dt: float) -> float:
    """指数関数的に重みを減衰"""
    return w * math.exp(-lam * dt)
```

`MetabolicControl.step()` は全エッジを走査し、上記の計算を適用します。

```python
class MetabolicControl:
    def step(self, graph: Graph, dt: float = 1.0) -> dict[str, int]:
        edges_removed = 0
        nodes_removed = 0

        for (u, v), edge in list(graph.edges.items()):
            # 保護ノードのエッジはスキップ
            if graph.nodes[u].is_protected or graph.nodes[v].is_protected:
                continue

            # 局所混雑度の算出
            C = graph.degree(u) + graph.degree(v)

            # 減衰率の算出
            lam = decay_rate(C, self.alpha, self.beta, self.gamma)

            # 重みの更新
            new_w = update_weight(edge.weight, lam, dt)

            if new_w < self.prune_threshold:
                graph.remove_edge(u, v)
                edges_removed += 1
            else:
                edge.weight = new_w

        # 孤立ノードの除去（保護ノードを除く）
        for nid in list(graph.nodes):
            if graph.degree(nid) == 0 and not graph.nodes[nid].is_protected:
                graph.remove_node(nid)
                nodes_removed += 1

        return {"edges_removed": edges_removed,
                "nodes_removed": nodes_removed}
```

## 7. 設計上の選択とその根拠

### 7.1 なぜ C = deg(u) + deg(v) なのか

和を使う理由は、エッジの「両端が混雑している」状態を捉えるためです。片方だけが混雑していてももう片方が疎な場合、そのエッジは疎な側のノードにとって重要な接続かもしれません。和を使うことで、両端が共に混雑している場合のみ高い混雑度となります。

### 7.2 なぜ指数関数的減衰なのか

線形減衰（w ← w - λ×dt）では、負の値になりうるため追加のクランプ処理が必要です。指数関数的減衰なら常に正値を保ちます。また、生物の放射性崩壊や薬物動態と同様の数理モデルであり、「残存量に比例して減衰する」という自然な直感に合致します。

### 7.3 なぜ α = 2 が推奨なのか

α = 1 (線形): 混雑度に対する応答が緩やか。混雑領域の整理が不十分になりがち。
α = 2 (二乗): 適度な非線形性。混雑領域を重点的に整理しつつ、疎な領域への影響を抑える。
α = 3以上: 混雑領域への応答が急激すぎ、パラメータ感度が高くなる。

α = 2は「混雑度が2倍で加速項が4倍」という直感的に理解しやすいスケーリングを提供します。

## 8. まとめ

代謝制御の要点をまとめます。

| 特性 | 内容 |
|---|---|
| **核心** | 局所混雑度C = deg(u) + deg(v)に基づく適応的な減衰 |
| **数式** | λ(C) = β(1 + γC^α)、w ← w × exp(-λ × dt) |
| **計算量** | O(1)/エッジ（局所統計のみ使用） |
| **性質** | 混雑→速い整理、疎→慎重な保持 |
| **分散** | エッジ間依存なし、パーティション分割可能 |
