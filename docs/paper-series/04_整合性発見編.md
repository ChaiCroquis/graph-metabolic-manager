## 1. はじめに

[前回（希少性保護編）](03_希少性保護編.md)では、二段階審査による希少ノードの保護を解説しました。Phase 1の保護期間中に「接続が見つかれば」ノードは生き残る、という設計です。

では、その「接続」はどうやって見つけるのでしょうか。

人間が一つ一つノード間の関係を調べるのは現実的ではありません。自動的に、しかも数学的に根拠のある方法で、**隠れた関連**を発見する仕組みが必要です。

それが**整合性発見**です。スペクトルグラフ理論のラプラシアン固有値を使って、直接接続されていないノード間の構造的な類似性を発見します。

## 2. 基本アイデア: 構造指紋

### 2.1 なぜ構造的類似性なのか

2つのノードが直接つながっていなくても、それぞれの「近傍の構造」が似ていれば、本質的に似た役割を担っている可能性があります。

例えば:
- 医療知識グラフで、ある希少疾患の治療薬と、別の疾患の薬剤が、それぞれの薬剤-疾患ネットワーク上で似た構造パターンを持つ → 薬剤リポジショニングの候補
- サプライチェーンで、2つの遠い仕入先が、それぞれの取引構造で似たパターンを持つ → 同じ脆弱性を共有する可能性

この「近傍構造の類似性」を、数学的に定量化するのがラプラシアン固有値スペクトルです。

### 2.2 ラプラシアン行列とは

グラフの隣接行列Aと次数行列Dから、ラプラシアン行列Lを定義します。

```
L = D - A
```

- D: 対角行列。D[i][i] = ノードiの次数
- A: 隣接行列。A[i][j] = エッジ(i,j)の重み（接続なしは0）

ラプラシアン行列の固有値は、グラフの構造的性質を反映します。

- **最小固有値**: 常に0（連結成分の数に関連）
- **第2固有値（代数的連結度）**: グラフの連結性の強さ
- **固有値の分布**: クラスタリング構造、直径、密度の情報

### 2.3 構造指紋の生成

各ノードの構造指紋を生成する手順は以下です。

```
1. ノードnを中心とするk-hop部分グラフを抽出
2. 部分グラフのラプラシアン行列Lを構築
3. Lの固有値を昇順にソート
4. 先頭dim個の固有値を「構造指紋」として取得
```

k-hop部分グラフとは、ノードnからkホップ以内で到達可能なノードとエッジの集合です。これにより、ノードの局所的な構造環境を切り出します。

```python
def compute_structural_repr(graph: Graph, dim: int = 8) -> np.ndarray:
    """ラプラシアン固有値による構造表現を生成"""
    L = graph.laplacian_matrix()  # D - A
    eigenvalues = np.linalg.eigvalsh(L)
    eigenvalues.sort()

    # 固定長dim次元に正規化
    if len(eigenvalues) >= dim:
        return eigenvalues[:dim]
    else:
        # 次元が足りなければゼロパディング
        result = np.zeros(dim)
        result[:len(eigenvalues)] = eigenvalues
        return result
```

## 3. 整合性スコア

### 3.1 三層構造のスコアリング

2つのノードの関連性を、3つの観点から評価します。

**第1スコア S_sys（系統的類似性）** — 重み: 7

構造指紋間の類似度です。以下の3つのサブスコアの組み合わせです。

- S_cos: 構造指紋ベクトルのコサイン類似度
- S_struct: 構造差分に基づく類似度（逆距離変換: 1 / (1 + ||a - b||)）
- S_sign: 固有値の符号パターンの一致率

```
S_sys = (S_cos + S_struct + S_sign) / 3
```

**第2スコア S_rel（関係類似性）** — 重み: 2

2つのノードの近傍集合の重複度（Jaccard係数）です。

```
S_rel = |N(r) ∩ N(c)| / |N(r) ∪ N(c)|
```

共通の隣接ノードが多いほど、関係的に類似していると判断します。

**第3スコア S_attr（属性類似性）** — 重み: 1

ノードの属性情報（タイプ、メタデータ）の一致度です。

```
S_attr = type_match * 0.5 + metadata_overlap * 0.5
```

### 3.2 複合スコアの合成

3つのスコアを重み付きで合成します。

```
S = (7 * S_sys + 2 * S_rel + 1 * S_attr) / 10
```

重み比 **7:2:1** の設計意図:

- **S_sys に最大の重み (7)**: 構造的な類似性が最も信頼できる指標。属性名の表記ゆれやメタデータの欠損に影響されない
- **S_rel に中程度の重み (2)**: 共通の隣接ノードは関連性の手がかりだが、グラフの密度に依存する
- **S_attr に最小の重み (1)**: 属性情報は有用だが、入力の品質に左右される

## 4. サンドイッチ閾値

### 4.1 なぜ上限があるのか

通常の類似度判定では「スコアが高いほど良い」とします。しかし、整合性発見では**上限閾値**を設けます。

```
採用条件:  theta_L <= S <= theta_U

推奨値:  theta_L = 0.70,  theta_U = 0.80
```

上限を設ける理由は**上げすぎ逆転の防止**です。

整合性スコアが過度に高い (S > theta_U) 場合:

1. **自明な関連**: 「りんごとりんごは似ている」は発見ではない。既に誰もが知っている関連をわざわざエッジとして追加する必要はない
2. **過剰結合のリスク**: 類似度が高すぎる全てのペアにエッジを張ると、グラフが過度に密になる。代謝制御が整理したものを整合性発見が復元してしまう矛盾が生じる
3. **ノイズの誤保護**: 実はゴミノード同士が偶然似た構造を持つケースでも、高スコアなら接続されてしまう

### 4.2 サンドイッチの意味

下限閾値と上限閾値で挟む（サンドイッチする）ことで、「適度に似ているが自明ではない」関連のみを発見します。

```
S < theta_L (0.70):  不採用（類似度不足。関連性が不十分）
     |
theta_L <= S <= theta_U:  採用（適度な類似性。発見の価値あり）
     |
S > theta_U (0.80):  不採用（上げすぎ逆転。自明または過剰）
```

### 4.3 閾値の設定指針

| パラメータ | 値 | 設定根拠 |
|---|---|---|
| theta_L | 0.70 | これ未満では構造的類似性が不十分 |
| theta_U | 0.80 | これ以上では自明な関連が増え、発見の価値が低下 |
| 幅 (theta_U - theta_L) | 0.10 | 狭い帯域により高精度な発見を実現 |

帯域が広すぎると偽陽性（本当は無関係なペア）が増え、狭すぎると発見件数が極端に少なくなります。0.10の帯域幅は、検証実験で有効な発見を安定的に生み出す値として確認されています。

## 5. 発見プロセスの全体像

```
入力: 希少ノードr, 候補ノード群 {c1, c2, ..., cN}
  |
  v
Step 1: 構造指紋の取得
  repr_r = get_repr(r)           # キャッシュあり
  repr_ci = get_repr(ci)         # キャッシュあり
  |
  v
Step 2: 各候補との整合性スコア算出
  for ci in candidates:
    S_sys = structural_similarity(repr_r, repr_ci)
    S_rel = relational_similarity(r, ci)
    S_attr = attribute_similarity(r, ci)
    S = (7*S_sys + 2*S_rel + 1*S_attr) / 10
  |
  v
Step 3: サンドイッチ閾値による採用判定
  if theta_L <= S <= theta_U:
    → 採用: ノードrとciの間にエッジを追加
    → ノードrの spoke_up フラグを True に設定
  else:
    → 不採用
```

構造指紋の計算はコストが高い（固有値分解が必要）ため、キャッシュを用いて再計算を回避します。

## 6. 実装

### 6.1 整合性スコアの算出

```python
def consistency_score(
    repr_r: np.ndarray,
    repr_c: np.ndarray,
    s_rel: float = 0.5,
    s_attr: float = 0.5,
    w_sys: float = 7.0,
    w_rel: float = 2.0,
    w_attr: float = 1.0,
) -> float:
    """複合整合性スコアを算出"""
    # 系統的類似性 S_sys
    s_cos = cosine_similarity(repr_r, repr_c)
    s_struct = structural_similarity(repr_r, repr_c)
    s_sign = sign_similarity(repr_r, repr_c)
    s_sys = (s_cos + s_struct + s_sign) / 3.0

    # 複合スコア
    total_w = w_sys + w_rel + w_attr
    score = (w_sys * s_sys + w_rel * s_rel + w_attr * s_attr) / total_w
    return score
```

### 6.2 発見の実行

```python
class ConsistencyDiscovery:
    def discover(self, graph: Graph,
                 rare_nodes: list[int],
                 candidate_nodes: list[int]) -> list[dict]:
        """希少ノードと候補ノード間の隠れた関連を発見"""
        discoveries = []

        for r in rare_nodes:
            repr_r = self._get_repr(graph, r)

            for c in candidate_nodes:
                if r == c:
                    continue
                if graph.has_edge(r, c):
                    continue  # 既存の接続はスキップ

                repr_c = self._get_repr(graph, c)

                # S_rel, S_attr を自動計算
                s_rel = relational_similarity(graph, r, c)
                s_attr = attribute_similarity(graph, r, c)

                score = consistency_score(
                    repr_r, repr_c, s_rel, s_attr
                )

                # サンドイッチ閾値判定
                if self.theta_l <= score <= self.theta_u:
                    discoveries.append({
                        "rare": r, "candidate": c,
                        "score": score
                    })

        return discoveries
```

## 7. 希少性保護との連携

整合性発見と希少性保護は密接に連携します。

```
希少ノードが Phase 1 に入る
  |
  v
整合性発見が毎ステップ実行
  |
  v
隠れた関連が発見された場合:
  1. ノード間にエッジを追加
  2. 希少ノードの spoke_up = True
  3. Phase 2終了時に「接続獲得」と判定 → 削除回避
```

この連携により:
- **保護が受動的でない**: Phase 1の間に能動的に接続を探す
- **発見に根拠がある**: 構造的類似性という数学的根拠に基づく
- **偽陽性を制御**: サンドイッチ閾値により、ゴミノードへの偽接続を抑制

## 8. ラプラシアン固有値の直感的な意味

なぜラプラシアン固有値が「構造指紋」として機能するのでしょうか。

固有値は以下のグラフ特性を反映します。

| 固有値の特徴 | 反映するグラフ特性 |
|---|---|
| 0の個数 | 連結成分の数 |
| 第2固有値の大きさ | グラフの連結性（代数的連結度） |
| 最大固有値 | 最大次数の上界 |
| 固有値の分布パターン | クラスタリング構造 |
| 固有値の間隔 | 構造の規則性 |

2つのノードの近傍が、連結性、クラスタリング構造、密度分布で似ている場合、それぞれの部分グラフのラプラシアン固有値も似た分布を示します。

これは「グラフのDNA」とも呼べるものです。直接的な属性を比較せず、純粋に構造的な特徴だけで類似性を判定できるため、メタデータの欠損や表記ゆれに影響されません。
