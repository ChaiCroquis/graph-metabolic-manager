## 1. 問題: 増え続けるグラフデータ

グラフ構造のデータは至るところにあります。

- ECサイトの商品推薦ネットワーク
- 企業のナレッジベースにおける文書間の参照関係
- 医療分野の薬剤-疾患-遺伝子ネットワーク
- SNSのユーザー間関係グラフ

これらに共通する課題は、**データが増える一方で減らない**ことです。

ノードは追加されるが削除されない。エッジは張られるが切られない。気がつけば、推薦グラフに10年前の季節商品が残り、ナレッジベースに廃止された社内規定が生き残っている。

「定期的に古いデータを消せばいい」。直感的にはそう思えます。しかし、単純な削除には致命的な落とし穴があります。

---

## 2. 従来手法の限界

代表的なデータ整理手法を比較してみましょう。

| 手法 | 原理 | 致命的欠陥 |
|---|---|---|
| **TTL** (Time To Live) | 一定期間後に削除 | 古くても価値あるデータを失う |
| **LRU** (Least Recently Used) | 最も長くアクセスされていないものから削除 | アクセス頻度と価値は比例しない |
| **閾値ベース削除** | 接続数が少ないノードを削除 | 希少だが重要なノードを殺す |
| **手動キュレーション** | 専門家が判断 | スケールしない、コストが高い |

いずれの手法にも共通する根本的な問題が3つあります。

**問題1: 低接続 ≠ 低価値**

ECサイトで考えてみましょう。あるニッチな専門書は、購入者が年に数人しかいない。推薦グラフ上の接続は極めて少ない。しかし、その数人にとっては唯一無二の商品であり、他では代替できない。閾値ベース削除はこのような「希少だが高価値」なノードを真っ先に殺します。

**問題2: 見えていない関連がある**

医療知識グラフで、ある希少疾患の治療薬と、まったく別の疾患の薬剤が、構造的に類似したネットワークパターンを持つことがあります。これは薬剤リポジショニング（既存薬の新適応）の候補かもしれません。しかし、直接的な接続がなければ、人間の目にもアルゴリズムにも見えません。

**問題3: 固定ルールは環境変化に追随できない**

グラフの密度は時間とともに変化します。初期は疎で、成長期は急激に密になり、成熟期は緩やかに変化する。固定パラメータの削除ルールは、ある時期には過剰に、別の時期には過小に作用します。

## 3. 提案: 4要素統合型の「代謝管理」

生物の身体には「代謝」があります。不要な細胞は自然にアポトーシス（細胞死）で除去され、必要な細胞は免疫系で保護され、新しい細胞間の連携が自然に形成される。そして、恒常性維持機構（ホメオスタシス）が全体のバランスを保つ。

この生物学的メカニズムをグラフデータ管理に応用したのが、本アルゴリズムです。

4つの要素が連携して動作します。

```
┌─────────────────────────────────────────┐
│          MetaControl（メタ制御）          │
│    グラフの「健康状態」を監視し、          │
│    代謝の強度を自動調整する               │
│                                         │
│              │ alphaを調整               │
│              v                           │
│  ┌─────────────────────┐                │
│  │ MetabolicControl    │                │
│  │ （代謝制御）         │ 混雑した領域を  │
│  │                     │ 重点的に整理    │
│  └─────────┬───────────┘                │
│            v                             │
│  ┌─────────────────────┐                │
│  │ RarityProtection    │                │
│  │ （希少性保護）       │ 希少ノードに    │
│  │                     │ 猶予を与える    │
│  └─────────┬───────────┘                │
│            v                             │
│  ┌─────────────────────┐                │
│  │ ConsistencyDiscovery│                │
│  │ （整合性発見）       │ 隠れた関連を    │
│  │                     │ 見つける        │
│  └─────────────────────┘                │
└─────────────────────────────────────────┘
```

### 要素1: 代謝制御（自動整理）

生物の代謝に相当する機能です。エッジの重みを局所的な混雑状況に基づいて減衰させます。

核心的な性質は**局所性**です。各エッジは自分の両端ノードの次数だけを見て減衰速度を決定します。混雑している領域のエッジは速く減衰し、疎な領域のエッジは緩やかに減衰します。グラフ全体を走査する必要がないため、O(1)/エッジで動作します。

→ 詳細は[第2回: 代謝制御編](02_代謝制御編.md)

### 要素2: 希少性保護（誤削除防止）

免疫系に相当する機能です。代謝制御だけでは、接続が少ないノードから先に孤立してしまいます。しかし「接続が少ない = 価値がない」とは限りません。

二段階の審査期間を設け、即座の削除を防ぎます。Phase 1では無条件に保護し、Phase 2で最終判定を行います。この間に整合性発見（要素3）が新たな接続を見つければ、ノードは生き残ります。

→ 詳細は[第3回: 希少性保護編](03_希少性保護編.md)

### 要素3: 整合性発見（隠れた関連の発見）

新しい細胞間結合の形成に相当する機能です。保護されたノードと既存のノードの間に、構造的な類似性がないかを調べます。

ラプラシアン行列の固有値スペクトルを「構造指紋」として利用し、直接接続されていないノード間の隠れた関連を数学的に発見します。

→ 詳細は[第4回: 整合性発見編](04_整合性発見編.md)

### 要素4: メタ制御（自動チューニング）

ホメオスタシス（恒常性維持）に相当する機能です。グラフの平均次数と目標値の乖離から「健全性指標」を算出し、代謝制御のパラメータを自動調整します。

→ 詳細は[第5回: メタ制御編](05_メタ制御編.md)

## 4. 設計上の主要な特性

### 4.1 局所性

全4要素が局所的な情報のみで動作します。代謝制御はエッジ両端の次数、希少性保護はノードの次数、整合性発見はk-hop近傍の部分グラフ、メタ制御は平均次数という集計統計量を使います。

これにより:
- **計算量がスケーラブル**: O(1)/エッジの代謝制御
- **分散環境で実行可能**: 各処理主体が局所統計のみで独立動作
- **逐次更新が可能**: バッチ処理に限定されない

### 4.2 適応性

固定パラメータではなく、グラフの状態に応じてパラメータが自動調整されます。環境変化に追随できるため、初期設定のチューニングが容易になります。

### 4.3 保護と整理の両立

代謝制御（整理）と希少性保護（保護）は一見矛盾しますが、本アルゴリズムでは保護集合に入ったノードのエッジは代謝制御の対象外となる設計により、両立を実現しています。

## 5. 実装と検証

リファレンス実装はPythonで約1,500行（テスト除く）です。

```python
from graph_metabolic_manager import Graph, GraphMetabolicManager

# グラフを構築
g = Graph()
n1 = g.add_node("Product A")
n2 = g.add_node("Product B")
n3 = g.add_node("Rare Insight")
g.add_edge(n1, n2, weight=1.0)
g.add_edge(n1, n3, weight=0.1)

# 全機能ONで100ステップ実行
mgr = GraphMetabolicManager(g, enable_meta=True, enable_rarity=True)
mgr.run(steps=100)
print(mgr.summary())
```
