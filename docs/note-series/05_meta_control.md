## 導入

体温は36.5度前後に保たれています。

暑いと汗をかき、寒いと震える。
誰かが「今37度だから冷やそう」と考えているわけではなく、
**システム自体が自分のバランスを取っている。**

情報のネットワークにも、同じことができないか？

---

## 「健全な状態」を定義する

グラフの健全性指標 H を、こう定義します：

```
H = 1 - |k_avg - k_opt| / k_opt
```

- **k_avg** : 現在の平均次数（実際の密度）
- **k_opt** : 最適な平均次数（目標）
- **H** : 0〜1。1に近いほど健全

k_avg = k_opt のとき H = 1.0（完全に健全）。
k_opt から離れるほど H は下がります。

---

## 自動調整のしくみ

H が目標値 H_target を下回ると、
代謝制御のパラメータ α が自動的に調整されます：

```
Δα = η × δk^n
```

- **η** : 学習率（どのくらい敏感に反応するか）
- **δk** : k_avg と k_opt の差
- **n** : 指数（4乗 = 大きな偏差に強く反応）

**過密（k_avg > k_opt）の場合**
H は低くなり、α が増加 → 枝刈り強化 → 密度低下

**過疎（k_avg < k_opt）の場合**
H は低くなり、α が減少 → 枝刈り緩和 → 密度維持

**最適（k_avg ≈ k_opt）の場合**
H は高く、α は変化なし → 現状維持

---

## 体温調節のアナロジー

**体温** → グラフでは **平均次数 k_avg**

**平熱 36.5度** → グラフでは **最適次数 k_opt**

**発汗・震え** → グラフでは **α パラメータの調整**

**「健康」の感覚** → グラフでは **Health Index H**

人間が何も意識しなくても体温が保たれるように、
このシステムも自動的に最適な状態を維持します。

## 4つの機能が揃った

1. **代謝制御** — 不要なつながりを自動的に減衰
1. **希少性保護** — 少数派に猶予期間を与える
1. **整合性発見** — 隠れた関係を数学的に検出
1. **メタ制御** — 全体のバランスを自動調整

4つが組み合わさることで、
「放っておいても勝手にバランスを取るグラフ管理システム」になります。
